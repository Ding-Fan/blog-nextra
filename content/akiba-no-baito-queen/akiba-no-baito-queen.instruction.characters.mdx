# キャラクター設定資料

## 概要

この資料は「アキバのバイト女王」に登場するキャラクターの詳細な設定情報を提供します。各キャラクターの外見、性格、話し方、役割について詳しく説明しています。

## 利用可能なキャラクター

### Character Component で使用できるキャラクター
- `tenchou` (店長) - Old otaku manager with cap and glasses
- `watashi` (私) - Young blonde girl learning Japanese
- `customer` (お客さん) - Store customer
- `hiromitsu` (浩三) - Regular customer, programmer

### アバターシステム
- **自動アバター**: Character の `name` prop に基づいて自動的に読み込まれます
- **デフォルトパス**: 
  - `tenchou` → `/images/characters/tenchou-avatar.png`
  - `watashi` → `/images/characters/watashi-avatar.png`
  - `customer` → `/images/characters/customer-avatar.png`
  - `hiromitsu` → `/images/characters/hiromitsu-avatar.png`
- **カスタムオーバーライド**: `avatar="/custom/path.png"` でデフォルトを上書き可能
- **アバター無効化**: `noAvatar={true}` で絵文字のみ表示
- **フォールバック**: アバター画像の読み込みに失敗した場合、絵文字アバターが表示されます
- **ゲームライクな表示**: アバターがダイアログボックスと一緒に表示され、没入感のある体験を提供

## 感情表現システム

キャラクターコンポーネントで利用可能な感情（日本語テキストタグ表示）:

### 基本感情
- `normal` - タグなし（デフォルト状態）
- `angry` - 怒り（赤色背景）
- `happy` - 嬉しい（黄色背景）
- `confused` - 困惑（グレー背景）
- `tired` - 疲れた（紫色背景）
- `excited` - 興奮（緑色背景 + バウンスアニメーション）
- `nervous` - 緊張（水色背景）
- `embarrassed` - 恥ずかしい（ピンク背景）

### 学習関連感情
- `thoughtful` - 考え中（深い青背景）
- `curious` - 好奇心（オレンジ背景）
- `understanding` - 理解した（明るい緑背景）
- `amazed` - 驚き（金色背景 + キラキラ効果）
- `grateful` - 感謝（温かい黄色背景）
- `proud` - 誇らしい（深い紫背景）
- `gentle` - 優しい（ソフトな青背景）
- `serious` - 真剣（ダークグレー背景）

### 技術実装仕様
- **表示位置**: アバター画像の右上角
- **フォントサイズ**: 12px（小さめで邪魔にならない）
- **背景**: 半透明の色付き背景（各感情に対応）
- **テキスト色**: 白色（読みやすさ優先）
- **角丸**: 4px（ソフトな印象）
- **アニメーション**: 
  - `excited` - 軽いバウンス効果
  - `amazed` - 微細なキラキラ効果
  - その他 - フェードイン効果（0.3秒）

## キャラクター使用例

### 基本的な使用方法
```jsx
{/* 自動アバター - 推奨（右上に「嬉しい」タグが表示）*/}
<Character name="tenchou" emotion="happy">
こんにちは！
</Character>

{/* 驚きの感情（右上に「驚き」タグ + キラキラ効果）*/}
<Character name="watashi" emotion="amazed">
すごいです！
</Character>

{/* 理解した瞬間（右上に「理解した」タグ）*/}
<Character name="watashi" emotion="understanding">
あ、わかりました！
</Character>

{/* アバターなし、絵文字のみ（感情タグは表示されない）*/}
<Character name="watashi" noAvatar={true} emotion="confused">
内心の独白...
</Character>
```

### アクションシーン
```jsx
<Character name="watashi" isAction>
nervously adjusts her blonde ponytail and the required cap
</Character>

<Character name="tenchou" isAction>
スマートメガネを調整しながらスマホモックアップを見る
</Character>
```

### 言語学習コンテキスト
```jsx
<Character name="watashi" emotion="confused">
あの...すみません、「ユニフォーム」って何ですか？
</Character>

<Character name="tenchou" emotion="tired">
制服だよ、制服。日本語まだまだだな...
</Character>
```

### Linux Tutorial コンテキスト
```jsx
{/* 好奇心を持って質問（右上に「好奇心」タグ）*/}
<Character name="watashi" emotion="curious">
../..って何ですか？どこに行くんですか？
</Character>

{/* 優しく説明（右上に「優しい」タグ）*/}
<Character name="hiromitsu" emotion="gentle">
いい質問ですね！..は「親ディレクトリ」という意味で、../..は「2つ上の階層」に移動します。
</Character>

{/* 理解の瞬間（右上に「理解した」タグ）*/}
<Character name="watashi" emotion="understanding">
あ、わかった！階段を2段上がるような感じですね！
</Character>

{/* 嬉しそうに反応（右上に「嬉しい」タグ）*/}
<Character name="hiromitsu" emotion="happy">
その通り！とても良い例えですね。
</Character>
```


## キャラクター一貫性のガイドライン

### 言語学習エピソード（メインシリーズ）
キャラクターは典型的な性格を維持します:
- **店長**: 少しせっかち、オタクの参照を使用、でも基本的に親切
- **私**: 学習に熱心、言語的ミスをする、言語への分析的アプローチ

### 技術学習エピソード（.linux、.excel、.software など）
専門的なキャラクターダイナミクスに焦点を当てます:
- **店長**: 忍耐強く、励ましてくれる教師になり、ユーモラスな比喩を使用
- **私**: 概念の理解に集中、明確化の質問をする
- **浩三**: 実践的な経験を共有し、現実的な問題解決を提供
- **コンテンツ**: 言語学習よりもスキル習得を優先
- **ユーモア**: エンゲージメントを維持し、記憶の定着を助けるために使用

#### Linux Tutorial Episodes での特別なガイドライン
- **説明のペース**: 
  - 複雑なコマンドの前に「なぜ必要か」を説明
  - 一つのコマンドごとに理解を確認
  - 視覚的な構造（tree）を多用
- **リアルな学習体験**:
  - 「私も最初は混乱した」という共感
  - 「よくある間違い」を事前に紹介
  - 成功体験を積み重ねる
- **実用性の強調**:
  - 「このコマンドは実際の仕事で...」
  - 「プログラマーは毎日使う」
  - 「覚えておくと便利」
- **安全性の教育**:
  - 危険なコマンドの警告
  - 「必ず確認してから実行」
  - バックアップの重要性

## プロット要素と秘密

**重要**: ストーリーには段階的に明かされるべきプロット秘密が含まれています:
- **店長のスマートグラス**: 普通のサングラスの後ろにスマートグラスを隠し、この技術を隠すためにスマホモックアップを見ているふりをしています。これは語り手によって早期に明かされるべきではありません - 後のエピソードでの発見として保持してください。

この包括的なキャラクター資料により、「アキバのバイト女王」シリーズの一貫性があり、教育的で魅力的なコンテンツが保証されます！
