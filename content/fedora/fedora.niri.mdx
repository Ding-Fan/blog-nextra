set `spawn-at-startup "niriswitcher"` to config
- https://github.com/isaksamsten/niriswitcher?tab=readme-ov-file#configuration

add this to niri config

```kdl
// in the binds
    // niri switch
    Alt+Tab repeat=false { spawn "niriswitcherctl" "show" "--window"; }
    Alt+Shift+Tab repeat=false { spawn "niriswitcherctl" "show" "--window"; }
    // Grave is the ` key
    Alt+Grave repeat=false { spawn "niriswitcherctl" "show" "--workspace"; }
    Alt+Shift+Grave repeat=false { spawn "niriswitcherctl" "show" "--workspace"; }

```

https://github.com/better-ecosystem/better-control

to enable audio output to monitor connected via HDMI or DisplayPort, you can use `pavucontrol` (PulseAudio Volume Control) to manage your audio devices.

In the pavucontrol window that opens, go to the Configuration tab.

You will see a list of your audio hardware. Find the one that corresponds to your graphics card (it will likely mention Intel, HDMI, or DisplayPort).

Next to that device, you will see a dropdown menu labeled Profile. It is likely set to "Off".

Click the dropdown menu and select a profile that includes "Digital Stereo (HDMI) Output" or "(DisplayPort) Output".

Now, switch to the Output Devices tab. You should see your monitor listed as a new audio output device.

You can set it as the default by clicking the checkmark icon (Set as Fallback).

Your system will now recognize and use your monitor for audio output.
